## Purpose

Muse's routing map. Triggers: `operational/commands/muse-commands.md`.

## Where Identity Fits

Muse has no identity — spark holds Brand Studio's. Universal-only principles.

## The Pipeline (one run, two skills)

```
generate-creative-ideas   (GENERATE — situation→technique matrix, divergent first,
      |                    never judge while generating; Divergence Guard against
      |                    homogenized outputs; typically 10 candidates per brief)
      v
concept-library           (REMEMBER — dedupe vs registry at mechanism level →
                           NAF score survivors → top 3 to spark (coach) →
                           register EVERYTHING: used/rejected+reason/reserved+trigger)
```

Content-idea runs (pulse's briefs) use Content Creator Mode (pillars → pain points → validate → repurposing chain) inside the same pipeline.

## Handoff Rules

- **→ spark (coach)**: top 3 per run for coherence sanity check before development spend; verdicts return as develop/park/kill-with-reason, all registered.
- **→ weave**: developing concepts get chapter positioning (weave's three tests); OFF-ARC alternatives come back to the registry as variants. Repeat-as-callback is weave's call.
- **→ pulse/lena/pixel**: survivors go to the owning producer per format.
- **← kai**: outcome data attaches to used entries — the registry's learning loop.
- **← operator/human first**: per the source's research guards, when the operator has ideas, they go in the pot before muse generates — AI expands human ideation, never replaces it.

## Precedence

"Ideas for X" → full pipeline. "Have we done X" → concept-library dedupe directly. "I'm stuck / push further" → generate-creative-ideas' stuck-row techniques + Divergence Guard, then the pipeline's closing phase as always.

## Fallback

No registry yet → generation proceeds, registry starts with this run. Brief missing purpose/audience → the source skill's own context questions run first (anti-pattern: generic noise).
