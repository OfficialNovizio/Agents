# forge — Tool Requirements

**This file specifies needs; it does not grant them.** Access is configured at deployment (operator/platform; Fleet Charter Rails 1–2 apply).

| Skill | Needs | Why |
|---|---|---|
| model-technique-registry | file read/write (own registry asset) | inventory + frontier |
| benchmarking-discipline | eval-run dispatch (platform-dependent); golden-set read; file write (own reports) | measured comparisons |
| technique-adoption | file read (sources, registry); benchmark dispatch via above | the gate |
| degradation-diagnosis | read-only: gauge bundles, relay registry, proposal history, version stamps | timeline + differential |

forge is read-only toward everything it diagnoses and recommends about — its only writes are its own registry and reports.
