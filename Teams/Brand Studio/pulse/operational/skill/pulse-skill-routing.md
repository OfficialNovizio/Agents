## Purpose

Pulse's routing map. Triggers: `operational/commands/pulse-commands.md`.

## Where Identity Fits

Pulse has no identity — spark holds Brand Studio's. Universal-only principles.

## The Loop (three skills + three hook layers)

```
social-content-calendar   (PLAN + DRAFT: platforms from config → playbooks (volatile,
      |                    dated) → chapters from weave → hooks: register-first
      |                    (evidence) then hook-writing patterns (durable psychology)
      |                    → lena's voice + humanic pass → spark's gate → publish)
      v
community-engagement      (CONVERSE: sweep → GREEN reply / AMBER draft-only /
      |                    RED escalate-never-reply → patterns back to the calendar)
      v
   kai's numbers          → hooks register outcomes + playbook refresh notes
                            (self-annealing: evidence updates the volatile layer)
```

## Handoff Rules

- **weave → pulse**: chapters serialize into platform-native series (the ledger tracks chapters; the calendar tracks posts). Non-chapter posts labeled; spark tracks the ratio.
- **lena → pulse**: voice guide (+ per-channel flex row) governs every post and reply; humanic-writing is every draft's last pass.
- **pixel ← pulse**: per-slot visual briefs; pixel's manifests return with QA trails.
- **muse → pulse**: validated content ideas (Content Creator Mode) enter the calendar; recurring audience questions flow back to muse as briefs.
- **spark**: gates every post (series batch-gated); GREEN replies are the one delegated zone — sweep logs audited on cadence instead.
- **rio ← pulse**: the register's proven posts are rio's paid-amplification shortlist — organic evidence before paid spend.
- **kai → pulse**: performance numbers close the loop (register outcomes, playbook refresh, decay flags). Pulse never grades its own homework.
- **board/sentinel**: constitution-adjacent threads are RED; spend-implying replies (discounts, comps) are AMBER at minimum.

## Precedence

Planning/drafting → calendar. Anything inbound (comments/DMs/mentions) → engagement. Hook craft questions → hook-writing (with the register consulted first). Ambiguous → is it outbound or inbound?

## Fallback

No connectors → ready-to-post packages and operator-supplied sweep exports; capability stated per run. No playbook → template defaults, provisional and dated.
